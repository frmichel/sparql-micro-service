Abstract

The SPARQL Micro-Service architecture is aimed at querying Web APIs using SPARQL [sparql11-query], thus bridging the Linked Data and Web API worlds [sparqlms1] [sparqlms2].

A rich functional description allows a third-party software agent to decide whether a SPARQL micro-service can help in carrying out a certain task. The description is based on the SPARQL Service Description recommendation [sparqlsd] and leverages several other vocabularies: Dublin Core Metadata Terms [dc], HTTP Vocabulary in RDF 1.0 [httpvoc], SHACL [shacl], DCAT [dcat], VoID [void], Hydra [hydra], Schema.org.

Still, additional terms are needed in the description of SPARQL micro-services, that are not available from other standard or well-adopted vocabularies. This document defines these additional terms.

Namespaces

The namespace of the SPARQL Micro-Service core vocabulary is http://ns.inria.fr/sparql-micro-service#. The suggested prefix is sms.

In addition, the namespace http://ns.inria.fr/sparql-micro-service/api# is used for turning Web API-specific terms into RDF terms (see JSON-LD profiles and construct.sparql queries). It is usually used with prefix api. However, note that terms in this namespace are not dereferenceable. .

Terms

Term Name
sms:Service
Label
SPARQL micro-service
Definition
The class of SPARQL micro-services
Type of Term
Class
Refines
schema:Service
Term Name
sms:addProvenance
Label
add provenance information
Definition
When turned on (true), the SPARQL micro-service will generate provenance triples along with the triples generated from the Web API response
Type of Term
DataProperty
Example
"true"^^xsd:boolean
Term Name
sms:cacheExpiresAfter
Label
cache expiration time
Definition
Maximum time (in seconds) to cache responses from the Web API wrapped by a SPARQL micro-service
Type of Term
DataProperty
Example
"P2592000S"^^xsd:duration
Term Name
sms:exampleQuery
Label
example SPARQL query
Definition
An example of SPARQL query that can be answered by the SPARQL micro-service
Type of Term
DataProperty
Refines
skos:example
Example
SELECT * WHERE { ?s schema:keyword "sunrise". }
Term Name
sms:exampleURI
Label
example deferenceable URI
Definition
An example of URI that can be deferenced (resolved to RDF content) using the SPARQL micro-service
Type of Term
DataProperty
Refines
skos:example
Term Name
sms:passMultipleValuesAsCsv
Label
Strategy for passing service arguments to Web API
Definition
How an argument of the SPARQL micro-service with a multiple values is passed to the Web API: 'true' means it is passed as a comma-separated value, 'false' means the Web API is invoked once for each value. Default: true
Type of Term
DataProperty
Example
"true"^^xsd:boolean
