Abstract

The SPARQL Micro-Service architecture is aimed at querying Web APIs using SPARQL [sparql11-query], thus bridging the Linked Data and Web API worlds [sparqlms1] [sparqlms2].

A SPARQL micro-services is described by means of a SPARQL Service Description (SD) document that leverages several vocabularies: SHACL [shacl], DCAT [dcat], VoID [void], Schema.org and Hydra [hydra]. This provides rich functional description that allows a software agent to decide whether a micro-service can help in carrying out a certain task.

This document defines additional terms needed in the description of SPARQL micro-services, that were not available from other standard or well-adopted vocabularies.

Namespaces

The namespace of the SPARQL Micro-Service core vocabulary is http://ns.inria.fr/sparql-micro-service#. The suggested prefix is sms.

In addition, the namespace http://ns.inria.fr/sparql-micro-service/api# is used for turning Web API-specific terms into RDF terms (see JSON-LD profiles and construct.sparql queries). It is usually used with prefix api. However, note that terms in this namespace are not dereferenceable. .
